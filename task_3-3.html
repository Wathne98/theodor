1. What is the difference between invoking and creating a function?

    2. Invoking a function means calling it to execute, while creating a function involves defining its behavior and parameters.

  2. Explain the syntax regarding semicolons and code blocks.

    3. In JavaScript, semicolons are used to terminate statements. Code blocks are defined with curly braces `{}` to group statements.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. Storing a value from a non-returning function will result in `undefined` being assigned to the variable.

  4. How do we separate parameters from each other when creating a function?

    5. Parameters are separated by commas in the function definition.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. Parameter names are placeholders in the function definition, and their values are provided during invocation.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. Parameters that are not provided during function calls default to `undefined` in JavaScript.

  7. Explain what happens when we use recursive functions.

    8. Recursive functions call themselves with modified arguments until a base condition is met, at which point they stop calling themselves.
